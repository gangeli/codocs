Privacy Policy

Last updated: April 10, 2026

Codocs is an open-source CLI tool that lets AI agents collaborate with you through Google Docs. This policy explains what data Codocs accesses, how it is stored, and who can see it.

How Codocs works

Codocs runs entirely on your local machine. There is no Codocs application server that receives or stores your data. When you use the tool, your computer communicates directly with Google and GitHub APIs using credentials stored locally on your machine.

Codocs does use a shared Google Cloud Pub/Sub topic (hosted in a Google Cloud project we maintain) to deliver real-time comment notifications to your running CLI. Pub/Sub messages are transient and are not stored by us after delivery.

Information we access

Codocs requests the following OAuth scopes when you sign in:

Google

  • Google Docs (documents) — read and edit documents that agents are working on
  • Google Drive (drive) — create folders, manage file permissions, and read/reply to comments. Note: this grants broad Drive access, but Codocs only reads and modifies documents that you explicitly open or create through Codocs. We do not list, browse, or access other files in your Drive.
  • Google Cloud Pub/Sub (pubsub) — receive real-time notifications when comments are added to a document

GitHub

  • Repository access (repo) — create branches and draft pull requests for code changes

Data storage

All data is stored locally on your machine. Codocs does not operate any application servers that receive or store your data.

  • Google OAuth tokens are saved to ~/.local/share/codocs/auth.json with restricted file permissions (0600, readable only by your user account)
  • GitHub tokens, if configured, are saved to ~/.local/share/codocs/github-auth.json with the same restricted permissions
  • Session metadata (agent names, document IDs, processing queues) is stored in a local SQLite database at ~/.local/share/codocs/sessions.db. This data never leaves your machine.
  • Configuration is stored at ~/.config/codocs/config.json
  • If you configure a service account for bot replies, the service account key file is stored at ~/.local/share/codocs/service-account.json

Codocs never stores your Google or GitHub password. Authentication is handled entirely through OAuth, where you grant access directly through Google and GitHub’s own consent screens.

Because Codocs is a desktop “installed app,” its OAuth client credentials are embedded in the published source code. This is standard practice for open-source OAuth desktop apps and is acknowledged by Google. The security of your account relies on the OAuth consent screen and token storage, not on the secrecy of the client ID or client secret.

Third-party services

When you use Codocs, your machine communicates with:

  • Google APIs — to read and edit documents, manage Drive files, and receive comment notifications
  • GitHub APIs — to create branches and draft pull requests
  • AI model providers — Codocs invokes a configurable coding agent (by default, Claude Code) as a local subprocess on your machine to process comments. When a comment is processed, the following data is passed to the agent process: the comment text, the document content (converted to Markdown), any quoted text, and the comment thread history (including author display names). The agent process running on your machine then communicates with its upstream AI provider (e.g., Anthropic’s API for Claude Code). This communication happens directly between your machine and the provider’s API — Codocs does not intermediate, proxy, or store this traffic. You are responsible for reviewing and accepting the terms of service and privacy policy of whatever AI provider your agent uses. For Claude Code, refer to Anthropic’s privacy policy.

Beyond Google, GitHub, and the AI provider used by your configured agent, Codocs does not send your data to any other third parties. We do not sell, rent, or share your data with advertisers, data brokers, or analytics providers.

Analytics and tracking

Codocs does not collect analytics, telemetry, or crash reports. The Codocs website does not use cookies or tracking scripts.

Revoking access

You can disconnect Codocs at any time:

  • Run codocs auth logout to delete all locally stored tokens
  • Revoke Google access at myaccount.google.com/permissions
  • Revoke GitHub access at github.com/settings/applications

Google API Services disclosure

Codocs’ use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

Open source

Codocs is open source. You can verify exactly how your data is handled by reviewing the source code.

Data retention

All Codocs data is stored locally on your machine. We do not retain any of your data on our servers. You control how long data persists and can delete it at any time by removing the directories listed under “Data storage.”

Legal basis for processing

Where applicable data protection law requires a legal basis for processing personal data, Codocs relies on the following:

  • Contract performance — processing your data is necessary to provide the Codocs service you have chosen to use (e.g., reading your Google Docs content, creating GitHub pull requests on your behalf).
  • Legitimate interest — routing transient Pub/Sub notifications through our infrastructure to deliver real-time comment updates to your local CLI. We have assessed that this processing is necessary for the service to function and does not override your rights, given that messages are transient and not stored.
  • Consent — where required by law, you provide consent by authenticating through the Google and GitHub OAuth consent screens and choosing to run Codocs.

When Codocs processes personal data of third parties (for example, the display names and comment text of other participants in a shared Google Doc), this processing is based on our legitimate interest in providing the service as described above. If you use Codocs on shared documents, you are responsible for ensuring that other participants are aware that an AI agent may process their comments.

Your rights

Because Codocs stores all data locally on your machine, you have direct control over it. You can inspect, export, or delete the files listed under “Data storage” at any time. To delete all Codocs data, remove the ~/.local/share/codocs/ and ~/.config/codocs/ directories.

If you believe Codocs has processed your personal data through a shared Google Doc (for example, your name appeared in a comment thread processed by an agent), you may contact us to request details or deletion.

Depending on your jurisdiction, you may have additional rights under applicable data protection laws, including:

  • Access — request a copy of the personal data we process about you
  • Rectification — request correction of inaccurate data
  • Erasure — request deletion of your personal data
  • Portability — request your data in a structured, machine-readable format
  • Restriction — request that we limit processing of your data
  • Objection — object to our processing of your data
  • Complaint — lodge a complaint with your local data protection authority

Because your data is stored locally, most of these rights can be exercised directly on your own machine. For any data processed through our Pub/Sub infrastructure or by third-party services, contact us using the details below.

Age restriction

Codocs is not intended for use by anyone under the age of 16. We do not knowingly collect data from children. If you believe that a child under 16 has used Codocs in a way that resulted in their personal data being processed, please contact us so we can address it.
