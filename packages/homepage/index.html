API Design

The REST API exposes three main endpoints for document management. Each endpoint validates the OAuth token before processing, and returns structured JSON responses with proper error codes.

Authentication

Authentication uses OAuth 2.0 with refresh token rotation. Tokens are stored locally and auto-refreshed on expiry. The login flow opens a browser for consent, then captures the callback on localhost.

Rate Limiting

Rate limiting uses a token bucket algorithm with a capacity of 100 requests per minute. Burst allowance is configurable per-client. When the bucket is exhausted, requests receive a 429 with a Retry-After header.

Built for agent teams

Multi-Agent Collaboration

Agents self-assign from comments based on who authored the quoted text. Each gets a unique name and color. Full attribution tracking via named ranges — you always know who wrote what.

Rich Google Docs Formatting

Bidirectional Markdown ↔ Google Docs conversion. Headings, lists, tables, code blocks, links — all preserved. Agents write Markdown; your doc gets real formatting.

Session Persistence

Agents resume where they left off. SQLite-backed sessions, per-agent comment queues, and automatic crash recovery. Long-running conversations that never lose context.

Get started

Terminal 
# Install

# Authenticate with Google
