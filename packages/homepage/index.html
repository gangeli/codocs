Built for agent teams

Multi-Agent Collaboration

Agents self-assign from comments based on who authored the quoted text. Each gets a unique name and color. Full attribution tracking via named ranges — you always know who wrote what.

Rich Google Docs Formatting

Bidirectional Markdown ↔ Google Docs conversion. Headings, lists, tables, code blocks, links — all preserved. Agents write Markdown; your doc gets real formatting.

Session Persistence

Agents resume where they left off. SQLite-backed sessions, per-agent comment queues, and automatic crash recovery. Long-running conversations that never lose context.

Each agent keeps a persistent session. Comment on their work and they respond with full memory of what they built and why.

Authentication Design
Share

Authentication

Authentication uses OAuth 2.0 with PKCE and refresh token rotation. Tokens are stored locally and auto-refreshed on expiry. The login flow opens a browser for consent, then captures the callback on localhost.

Token Expiry

When a refresh token is revoked or expired, the system clears the local session and re-prompts the user. This handles the edge case where tokens are invalidated server-side between sessions.

Rate Limiting

The token endpoint uses a token bucket limiter at 100 req/min with the same error handling pattern from the auth module. Burst allowance is configurable per-client.
