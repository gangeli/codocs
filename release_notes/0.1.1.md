# v0.1.1

A bugfix release focused on **comment-listener uptime** and **comment-anchor robustness**. Long-running `codocs serve` sessions previously stopped responding to comments after a few hours; v0.1.1 traces that to a 4-hour TTL on Workspace Events subscriptions, fixes the renewal cadence, and adds a self-healing recreate path so a missed renewal doesn't take the listener down. Also picks up several anchor-matching fixes from the live-Drive end-to-end work.

## Fixes

- **Comment listener no longer goes silent every few hours.** Workspace Events caps the subscription TTL at 4 hours when `payloadOptions.includeResource: true` (which we need so the parser can recover the comment ID). Renewal was scheduled for every 6 days, so the upstream subscription always expired first and Drive stopped publishing — manifesting as repeated "health check: idle" reconnects with no actual messages flowing. Renewal now runs every 3 hours (well under the 4-hour cap), and an in-process recreate fires automatically if the upstream subscription disappears anyway, so recovery no longer requires restarting `codocs serve`. (`8353abb`)
- **Reconnect logs now report the real silence gap.** The watchdog reset `lastActivityAt` before scheduling the reconnect, so the value carried into `onReconnect` measured the backoff delay instead of the actual idle window — every idle reconnect logged `idle=1.0min` even though the trigger threshold is 30 minutes. Captured at the moment of trigger and threaded through. (`39b47fa`)
- **Comment anchors resolve through inline markdown markers.** When a comment's anchor sits inside `_emphasis_`, `**bold**`, or other inline marker pairs, the splice path now strips markers via an mdast walk (replacing a brittle regex) and falls back to a stripped-text match when the literal anchor straddles markers. Fixes the CA15 / CA20 / CA21 / CA22 regression suite end-to-end against live Drive. (`a0fd729`, `b7a4f6b`, `17ba8f2`, `600d2a5`, `458ff1d`)
- **Anchored paragraph splits no longer corrupt the doc.** The runner now bails on a split that would orphan an anchor, and the concurrent-edit check fires before the split rather than after. (`fda45a0`)
- **Multi-anchor sections handled correctly.** Section restore is skipped on whole-section deletes, which previously could resurrect deleted content. (`59c4a17`)

## Observability

- **Comment-listener health now hits the file log.** Adds an `info` callback so every Pub/Sub message arrival, skip reason (not-a-comment / bot self-reply), subscription `expireTime` at startup and on each renewal, and reconnect diagnostics (`lastActivityAgoMs`, `wasOpen`) are persisted at info level. Without this, silent filtering was indistinguishable from upstream silence. (`da21ec5`)

## Internal

- Root-level `vitest.workspace.ts` so `npx vitest run` from the repo root picks up each package's local config (e.g. the `__BUILD_VERSION__` define) instead of failing to load `serve.test.ts`. (`347224f`)
