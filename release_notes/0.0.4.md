# v0.0.4

The first substantial post-plumbing release. v0.0.1–v0.0.3 existed mostly to get the publish pipeline green; this one is 44 commits of actual product. Highlights: three new agent runners, chat tabs, Mermaid rendering, native Docs checkboxes, session resumption, a repair module, and a shift to user-identity replies by default.

## Highlights

### New agents and runner polish
- **Cursor agent runner** (`7749122`) — drives `cursor-agent` with print mode, resumable sessions, and the current Cursor model lineup.
- **Codex and OpenCode runners** (`e2083b2`) — OpenAI Codex CLI with approval-mode mapping, and OpenCode wired to Cerebras + GLM. Introduces a `RunnerCapabilities` contract so the settings panel now renders per-runner controls instead of Claude-specific ones.
- **Per-runner default model setting** (`a7c413a`) — pick a default model per agent type from the TUI; flows through orchestrator → runner → `--model`.
- **Harness subscription/quota detection** (`aeccfc1`, `6d4fcb3`) — detects which agents you have access to, reads billing tier and quota headroom, and ranks agents by preference on startup.
- **Concurrent forked sessions per comment** (`b6f39cf`) — unrelated threads assigned to the same agent now run in parallel via `claude --resume <base> --fork-session`, instead of serializing behind a per-agent queue. Follow-ups on an existing thread still resume the thread's linear session. Runners without fork support fall back to the legacy drain.

### New document features
- **Chat tabs** (`4dd8d76`) — fork a comment thread into its own Google Doc tab for multi-turn conversations. A new router decides whether a comment continues a chat or stays on the original thread; the `[MODE: chat]` classifier can auto-fork complex conversations. Built on googleapis v171's native tab API.
- **Mermaid diagrams** (`0d7268a`, `8f4221e`) — ` ```mermaid ` code blocks render to PNG and insert as inline images. Original source is stashed in SQLite and restored on the way back to Markdown via positional matching.
- **Native Google Docs checkboxes** (`277a91b`) — GFM task-list syntax (`- [ ]` / `- [x]`) round-trips through Google Docs' `CHECKBOX` bullet preset.

### Session and server lifecycle
- **Session resumption** (`ba0adb2`) — every session gets a 6-char ID. Resume with `codocs --resume <id>`, `codocs --resume` (most recent), or from the Welcome screen's new "Resume" list. Quit prints a copyable resume hint.
- **Drive-backed server lock** (`a001c78`, `e21abe4`) — a heartbeat in Drive `appProperties` prevents two codocs servers from attaching to the same doc. Works across machines. Heartbeat is compact (`cl_hb: {epoch}:{hash}`, well under the 124-byte limit), refreshes every 15s, and releases on shutdown.
- **`--silence-hook` and `--meta` flags** (`8ecbdd2`) — run a shell command when agents go idle, or self-rebuild-and-restart codocs from within itself. Idle detection is debounced on the busy→idle transition.

### Reply identity shakeup
- **User-identity replies by default** (`8f3bc94`) — replies now come from your own Google OAuth identity with a 🤖 prefix, so you can tell codocs's replies from yours at a glance. The bundled service-account key and `SECURITY.md` are gone. Opt in to a separate service-account identity with `--service-account [path]`.
- **Reply-loop prevention** (`03443d0`) — when replying as the user, a `ReplyTracker` records every reply ID codocs posts so the listener skips self-triggered events rather than re-enqueueing them forever.

### Operational tooling
- **`codocs repair` + startup health checks** (`43639ca`) — a Check/Issue/Fix registry runs scoped checks at startup and surfaces malformed doc IDs, missing auth, stale queue items, and dead sessions as fixable issues instead of hard exits. Auto-rechecks after every fix.
- **`codocs login` alias** (`51b1b08`) — top-level shortcut for `codocs auth login`. Error text now points at the shorter form.
- **Dynamic build version** (`ecab59d`) — `--version` and the welcome footer report the release version for tagged builds and a git hash (with dirty-diff fingerprint) for dev builds.

## Fixes
- **`--meta` respawn could deadlock on its own heartbeat** (`a1d21a4`) — the restart path now stops the heartbeat refresh timer *before* clearing the lock, so the child doesn't race the parent's dying renewal loop and bail with "duplicate server".
- **Out-of-bounds `insertText` on long docs** (`c43062f`) — replaced a 1:1 md-offset/doc-index assumption with proper interpolation between bracketing map entries, plus local-ratio extrapolation past the last entry. Fixes drift that accumulated to ~1.7K characters over a 30K-index doc.
- **Exit screen regressions** — resume command now actually appears after TUI quit (`777b474`, `5bd624f`), exit-screen crash on teardown errors (`2a666af`, `11ca977`), and the farewell now shows even if `waitUntilExit` rejects.
- **Malformed doc ID** (`7b89d24`) — the server-lock check no longer warns on malformed IDs (repair handles them), and there's now a "Quit" fix option when the bad ID came from the command line.
- **GitHub device-code 502s** (`8c9a97e`) — retries up to 3 times with 2s/10s/30s backoff before failing.
- **Permission accumulation when using a service account** (`b33107b`) — commenter access is revoked on shutdown, and the bot verifies the user still has access before posting on their behalf.
- **New docs open in pageless format** (`a4af40b`) — better for collaborative editing.
- **Cursor runner actually works** (`6fea29a`) — fixed binary name, print-mode flags, positional prompt, and `--resume` wiring.
- **Narrow pencil in the welcome menu** (`29970c2`) — swapped half-width `✎` for full-width `✏️` so the icon aligns with the rest of the menu.

## Internal
- **`/release` slash command** (`38866a9`) — prepares version bumps and `release_notes/<version>.md`. CI still drives publish, now reading body from the release notes file.
- **Full test suite green** (`db6f36d`) — replaced vitest's `vi.waitFor` (unimplemented in bun) with a local poll helper, and cancelled leaked idle-debounce timers in test teardown so the cross-file "Database closed" cascade is gone. Suite went from 255/16 to 313/0.
- **`make check` rebuilds first** (`9e0a8ea`) — cross-package typecheck resolves `@codocs/*` through built `dist/`, so stale output was masking real source changes.
- **`AgentType` union** (`6fea29a`) — `'claude' | 'codex' | 'cursor' | 'opencode'` replaces open strings throughout; `isAgentType()` validates CLI input at entry.
- **Legal/marketing** — logo, GitHub app, initial Terms + Privacy Policy (`7743d6c`), with follow-up revisions (`aa3f23e`, `18768bb`).
- Build fixes from the pre-0.0.3 scramble (`bcb6af6`, `3f59877`) and various small touches to the auth success page and prompts.
