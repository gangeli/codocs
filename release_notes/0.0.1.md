# v0.0.1

Initial public release of **codocs** — a CLI and core library that lets coding agents communicate with users through Google Docs. Comments in the doc become the agent's inbox; edits become its output. This release lands everything required to authenticate with Google, listen for Drive comment events, and run an agent harness end-to-end.

## Highlights

- **Core library (`@codocs/core`)** — bidirectional Markdown ↔ Google Docs conversion, agent attribution via named ranges, Drive comment management, Workspace Events subscription lifecycle, Pub/Sub pull listener, and a Claude-runner agent orchestration harness (`9269605`).
- **CLI (`@codocs/cli`)** — OAuth2 login, `read` / `edit` / `insert` / `sections` / `comment` commands, and a default `serve` command that wires Workspace Events to Pub/Sub and drives the agent (`9269605`).
- **Terminal UI built on Ink** with a round of ergonomic polish on the interactive surface (`4cb4e5b`).
- **Distributable binaries** for the CLI packaged alongside iteration on comment handling (`772df9a`).
- **Reply-to-comment support** so the agent can continue a thread instead of just posting new comments (`57415c7`).
- **Git worktree beta** for isolating agent runs (`143a3ef`).
- **Project homepage + e2e scripts** for smoke-testing the full flow and giving the project a public landing page (`3498e3b`, `ab76c30`, `f678c29`, `cf7c0fa`).
