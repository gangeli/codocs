# v0.1.0

The headline of this release is **listener reliability**: long-running codocs sessions used to silently stop responding to comments after a few hours when the underlying gRPC streaming-pull connection went half-open. v0.1.0 wraps the subscription in a self-healing manager â€” auto-reconnect on close events, watchdog that catches silent failures, and an idle-recycle guard with a sensible default. Validated against a 2-hour live soak on real Pub/Sub. The release also picks up a `--force-unlock` CLI flag, doc-anchor safeguards, smarter auto-restart, and a batch of eval improvements.

## Highlights

- **Self-healing Pub/Sub listener.** The streaming-pull connection sometimes silently dies (refresh-token expiry, half-open TCP after laptop sleep, transient gRPC errors the library swallows) without firing `close` or flipping `isOpen`. The listener now reconnects with backoff on the close event, watchdogs `subscription.isOpen`, treats stream errors as recovery signals, and recycles the connection after a 30-minute idle window by default. Reconnects surface through `onReconnect` so the TUI/log can show them. (`edab576`, `bafbbe4`)
- **`--force-unlock` CLI flag.** Bypasses an existing server lock when you know the previous codocs is gone (e.g. after a crash) without having to hand-edit Drive `appProperties`. (`27134cb`)
- **Smart auto-restart on code changes.** Codocs now auto-restarts itself when the watched code actually changes, instead of only when explicitly asked. (`222f26a`, `57312c1`)
- **Comment-anchor safeguards.** Before applying a doc edit driven by a user comment, codocs now validates that the comment's anchor still resolves cleanly â€” protecting against silent corruption when the anchor has drifted. (`3f23fd9`)

## Fixes

- Migration cleanup that was preventing some upgrades from completing. (`d2539d0`)
- The fallback agent resolver is now version-controlled and properly tested. (`fabd8c3`)
- `make` runs `npm install` automatically so first-time builds Just Work. (`164a40a`)

## Internal

- New `make e2e/connection` target with two modes: a ~3-min cycle test that forces real reconnects against the live Pub/Sub stream, and a `--soak-ms=N` mode that mirrors prod settings and verifies post-soak delivery after hours of idle. The 2-hour soak passed cleanly with 3 watchdog reconnects and zero errors. (`ea18e8c`, `15a6573`, `18cb682`)
- `listener-longevity.test.ts` covers four failure modes (loud close, watchdog half-open, silent error, silent half-open) over hours of fake-timer time.
- Tighter eval rubrics across bug-fix, doc-only, edge, follow-up, and QA cases; added behavior probes for stage/event-keyed structured logs, the no-warning bare `serve`, dropped empty rows, and multi-line `DELETE-WHERE`. Soft doc guards on permissive QA cases prevent hallucinated rewrites from passing. Prompt snapshots added for regression tracking. (`8b33c8e`, `c4d88ee`, `5cf8923`, `3448271`)
- Design-doc plumbing tweaks for the from-codebase generator. (`a57b67b`, `ffb7ca8`, `b7c10a8`)
