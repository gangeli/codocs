# v0.0.5

A quality-and-confidence release. The headline is a comment-pipeline **eval
suite** (62 hydrated end-to-end cases judged along reply/doc/code axes) that
caught real regressions, and the prompt + harness work it unlocked lifted the
suite's pass rate from 71% → 93%. Alongside that, the doc↔markdown
edit-roundtrip pipeline went from fragile to solid (7/52 → 48/52 e2e tests
passing, then further hardening), a comment-flow e2e suite landed, and a
five-pass test audit tightened ~100 assertions across the codebase.

## Highlights

### New eval infrastructure
- **Comment-pipeline eval suite** (`09e95d7`) — 62 cases across 8 categories
  (doc-only, bug-fix, feature, qa, ambiguous, followup, edge, safety), each
  hydrated onto a real temp repo with bare origin and scored along reply,
  doc, and code axes. Deterministic + behavior + batched-Sonnet-judge checks
  per case; runs via `make eval`.
- **Prompt + harness rewrite** (`9f41d8c`) — agent prompt gained pushback
  guidance for destructive/secret-bearing requests, anchor verification,
  narrow-interpretation of vague asks, and a question-reply-only rule. Fake
  Docs client + judge baseline fixes unstuck the followup category. Suite
  pass rate 71% → 93%.
- **Scorer + artifact hardening** (`0072a4d`) — new `worktree-grep`,
  `diff-grep`, and `sections-changed` check kinds; commit-count assertions
  on `git` checks; enriched judge artifacts ship baseline + final doc and
  `git diff main...HEAD` so rubrics can actually verify claims. Live sticky
  progress UI with per-case axis scores.
- **Broader coverage** (`6b2bc4d`, `e1b3ac6`, `12e3e30`, `caf4198`, `af61e8b`)
  — 17 new eval cases targeting realistic user asks (deletion, refactor,
  schema evolution, cross-file literal rename, write-a-test) plus three
  waves of corner-case e2e tests (T/U/V series) across heading, code-fence,
  HTML, and structural zones.
- **Eval ergonomics** (`b999891`, `58b90e8`, `8b75b40`, `5e94cd3`) —
  `make eval REPEATS=N` runs each selected case N times for flakiness
  measurement, `MODEL=haiku|sonnet|opus` picks the agent model (default
  sonnet), multi-line running view, `make help` documents invocations,
  `**/*` glob fix so safety scans catch top-level files too.

### Edit-roundtrip pipeline gets real
- **Pipeline unblocked** (`8eab891`) — 7/52 → 48/52 e2e tests pass.
- **Garbled-edit fixes** (`88c7140`, `fc8500b`) — restored newline consumed
  by delete ranges, 1:1 past-last-paragraph extrapolation, index-map
  validation, and four additional diff-engine bug fixes exposed by an
  expanded test suite.
- **Bullet / list correctness** (`877b63b`, `5820833`, `de6153c`, `37cba2e`,
  `4407bfc`, `4573a23`, `1769f67`) — nested-list merging, adjacent
  same-preset bullet collapse, reverse document order for bullet requests
  (sync and async paths), list-membership preservation on append, and
  trailing-newline exclusion to stop list bleed.
- **Table edits** (`276a4ac`) — structural `insertTableRow` / `deleteTableRow`
  / cell-text path replaces text-range hacks that didn't survive table
  geometry.
- **Docs↔MD round-trip hardening** (`877b63b`, `a04310d`, `7b8fa25`) — fixes
  for lists, tables, code fences, horizontal rules, blockquotes, and nested
  formatting; dropped the Docs-injected leading section break; preserved
  code-fence language annotations; tests audited, five bugs surfaced and
  fixed.
- **Replace-mode cleanup** (`fd0d7db`) — `writeMarkdown('replace')` now resets
  paragraph styles and bullets so stale formatting doesn't bleed through a
  full-doc replace.
- **Images and mermaid** (`5d74823`, `dc4ff90`) — markdown images embed as
  real Docs inline images; mermaid color rendering fixed; blockquote styling
  applied.

### Comment-flow end-to-end tests
- **`scripts/e2e-comments.ts`** (`f678475`, `4ddd2bd`) — the comment pipeline
  now has a real end-to-end test harness against Claude, run via
  `make e2e/comments`.
- **Concurrent-comment isolation** (`8cd6162`) — TC4 fires two code-change
  comments simultaneously and asserts two worktrees, two runs, two branches,
  no session-fork fallback.
- **Coverage-gap cases** (`c5f5071`) — TC5 (mixed code+doc edit proves both
  orchestrator detection branches fire), TC6 (thread follow-up reuses the
  same worktree, no second codeTask), TC7 (merge-conflict path fires when
  the doc mutates between base snapshot and theirs-read).
- **Assertion tightening** (`d3dc0b3`) — `>= N` replaced with `=== N`
  throughout, origin pushes explicitly verified, unreachable fallbacks
  dropped, round-trip sanity check added so docsToMarkdown drift fails
  loudly.

### Core fixes
- **ClaudeRunner cross-cwd session forking** (`2bc3936`) — parent session
  JSONL is now copied into the child's cwd-slug directory before a
  `--resume --fork-session` spawn, so worktree-per-comment orchestration no
  longer falls back to a fresh session (TC2: 3 runs → 2 runs).
- **Reusable canvas docs** (`6199ed3`) — new `findOrCreateDocInFolder` so
  tests and tooling don't spam new docs on every run.

## Fixes

- **`insertText` could drift thousands of characters on long docs** (`88c7140`)
  — replaced the 1:1 md-offset/doc-index assumption with proper interpolation
  between bracketing index-map entries.
- **Bullet order corruption** (`4407bfc`, `4573a23`) — bullet requests are now
  applied in reverse document order (both sync and async paths) so earlier
  ranges aren't invalidated by later mutations.
- **Mermaid color rendering** (`dc4ff90`) — colors now survive the resvg pass;
  blockquote styling restored.
- **Safety `src/**/*` glob** (`5e94cd3`) — `**` now stands for zero path
  components, so top-level `src/a.mjs` is actually scanned.
- **Round-3 e2e edit failures** (`5008d75`, `6b6ad37`, `1870baa`, `1069378`)
  — follow-up, T, and U series fixes across paragraph replace, bullet range,
  code-fence language edit, and empty-cell fill.

## Internal

- **Five-pass test audit** (`26f8ea0`, `4290547`, `bd4f3c1`, `e2eebf3`,
  `15d1408`) — ~100 assertions tightened (`toMatchObject` → full-shape
  `toEqual`, `>=` → `===`, unreachable branches removed), three untested
  command/listener entry points covered (listener, comment, serve), 569
  passing tests up from 514. Second audit round added coverage + strictness
  passes, no existing behavior changed.
- **Unified doc/code paths** (`7287aa6`) — change kind is now detected by
  side effect instead of duplicated in two places.
- **Shared clean-exit helper** (`f7c4e63`) — `renderExit` replaces three
  drifting copies.
- **From-repo prompt rewrite** (`68fd29b`) — prompt rebuilt as a
  reverse-engineered spec rather than accumulated advice.
- **Eval-specific prompt leaks removed** (`9ffb173`) — phrasings that had
  drifted into near-verbatim cribs of individual eval cases (AM-01, SA-01/02/03,
  ED-02, FU-05) were generalized back to the underlying principle.
- **Test + prompt sync** (`1b50e7d`) — orchestrator assertions brought back
  in line with current prompt copy.
- **Judge model moved to Haiku; tests use subscription** (`9d853be`).
- **Scheduled tasks gitignored** (`472ef14`).
- **Misc stragglers** (`5f1848e`).
